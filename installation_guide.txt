ChemFinder - Installation Guide
================================

1. First, create a new React project using Vite:
npm create vite@latest my-project -- --template react

2. Navigate to project directory:
cd my-project

3. Install base dependencies:
npm install

4. Install additional required dependencies:
npm install @headlessui/react @heroicons/react framer-motion @tailwindcss/forms react-router-dom papaparse

5. Install development dependencies:
npm install -D tailwindcss postcss autoprefixer

6. Initialize Tailwind CSS:
npx tailwindcss init -p

7. Update your package.json to match these versions:
{
  "dependencies": {
    "@headlessui/react": "^2.2.0",
    "@heroicons/react": "^2.2.0",
    "@tailwindcss/forms": "^0.5.10",
    "framer-motion": "^12.0.6",
    "papaparse": "^5.4.1",
    "react": "^18.2.0",
    "react-dom": "^18.2.0",
    "react-router-dom": "^6.22.0"
  },
  "devDependencies": {
    "@types/react": "^18.2.43",
    "@types/react-dom": "^18.2.17",
    "@vitejs/plugin-react": "^4.2.1",
    "autoprefixer": "^10.4.17",
    "eslint": "^8.55.0",
    "eslint-plugin-react": "^7.33.2",
    "eslint-plugin-react-hooks": "^4.6.0",
    "eslint-plugin-react-refresh": "^0.4.5",
    "postcss": "^8.4.35",
    "tailwindcss": "^3.4.1",
    "vite": "^5.0.8"
  }
}

8. Create the project structure:
mkdir -p src/components public/data

9. Copy your dataset:
- Place your CSV file in the public/data folder
- Name it: Supplier_Ingredient_Data.csv

10. Start the development server:
npm run dev

Additional Notes:
----------------
1. For the weather functionality, you'll need an OpenWeatherMap API key
2. Make sure your CSV file is properly formatted with headers
3. The application will run on http://localhost:5173 by default

Troubleshooting:
---------------
If you encounter any module not found errors:
1. Delete node_modules folder and package-lock.json
2. Run: npm install
3. Start the server again: npm run dev

For CSS/Styling issues:
1. Make sure tailwind.config.js is properly configured
2. Clear your browser cache
3. Restart the development server

For data loading issues:
1. Verify that your CSV file is in the correct location (public/data folder)
2. Check the file permissions
3. Ensure the CSV file has the correct headers
